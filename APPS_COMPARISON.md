# 🎯 ASCII RTX v1 - Сравнение трёх приложений

## Краткое сравнение

| Параметр | GUI 🖼️ | Interactive 🎮 | Batch 📊 |
|----------|---------|-----------------|---------|
| **Тип интерфейса** | Окно с меню | Консоль | Текст |
| **Управление** | WASD + Меню | WASD только | - |
| **Конфигурация** | Dropdown меню | config.ini | config.cpp |
| **Визуализация** | В реальном времени | В реальном времени | 500 кадров |
| **FPS** | ~60 | ~60 | Зависит |
| **Автосохранение** | Да (config.ini) | Нет | - |
| **Сложность** | Высокая | Средняя | Низкая |
| **Рекомендуется для** | Начинающих | Энтузиастов | Тестирования |

---

## 🖼️ GUI Приложение (ascii_rtx_gui.exe)

### Запуск
```powershell
.\run_gui.bat
# или
.\build\ascii_rtx_gui.exe
```

### Технология
- DirectX 11
- Dear ImGui
- Windows API

### Преимущества ✅
- Красивый интерфейс
- Выпадающие меню (dropdowns)
- Ползунки для параметров (sliders)
- Автосохранение конфигурации
- Полноэкранный режим
- V-Sync переключение
- Выбор разрешения прямо в приложении
- Профессиональный внешний вид

### Недостатки ❌
- Самый сложный в разработке
- Требует DirectX 11 и ImGui
- Может быть медленнее на слабых ПК

### Идеально для
- 👤 Новичков (всё интуитивно)
- 🎨 Людей, которые любят красивые интерфейсы
- 🔧 Экспериментирования с параметрами
- 📊 Демонстрации

### Файлы
```
src/apps/gui/
├── main.cpp           (540+ строк)
├── app_config.h/cpp   (конфигурация)
├── ascii_renderer.h/cpp (рендеринг)
└── imgui_setup.h/cpp  (меню)
```

## 🎮 Interactive Console (ascii_rtx_interactive.exe)

### Запуск
```powershell
.\build\ascii_rtx_interactive.exe
```

### Технология
- Windows Console API
- SDL2 (начально планировался, но упрощено)
- Неблокирующий ввод (GetAsyncKeyState)

### Преимущества ✅
- Простая консоль (нет зависимостей от графики)
- Быстрый ввод (реальное время)
- Показывает координаты камеры
- FPS счётчик
- Минималистичный

### Недостатки ❌
- Обновление ASCII медленнее (консоль медленнее, чем окно)
- Нет меню конфигурации
- Сложнее читать на высоких скоростях

### Идеально для
- 🎮 Игроков (сфокусировано на управлении)
- 💻 Linux/старых систем (теоретически)
- 🧪 Тестирования физики камеры
- 📐 Быстрого прототипирования

### Файлы
```
src/apps/interactive/
├── main.cpp           (200+ строк)
├── console_ui.h/cpp   (консоль)
└── input_handler.h/cpp (ввод)
```

## 📊 Batch Renderer (ascii_rtx_viewer.exe)

### Запуск
```powershell

# или

```

### Технология

- Быстрое тестирование логики
- Легко добавлять новые функции
- Статистика точна

### Недостатки ❌
- Нет интерактивности
- Просто список цифр
- Не увидите визуализацию в реальном времени

### Идеально для
- 🔬 Разработчиков (отладка)
- 📈 Анализа производительности
- 🤖 Автоматизации
- 📝 Получения статистики

### Файлы
```
src/apps/viewer/
└── main.cpp (170+ строк)
```

## 🆚 Прямое сравнение

### GUI vs Interactive
```
GUI:
+ Красивое меню
+ Выбор разрешения в приложении
+ Полноэкранный режим
+ Автосохранение конфига
- Сложнее использовать на слабых ПК

Interactive:
+ Прямолинейное управление
+ Сфокусировано на WASD
+ Минимум внешних зависимостей
- Нет визуального меню
```

### GUI vs Batch
```
GUI:
+ Интерактивно
+ Видите результат сразу
+ Можно играть

Batch:
+ Просто получить результат
+ Быстро запустить тест
+ Подходит для автоматизации
```

## 📋 Рекомендации по использованию

### Случай 1: "Я хочу просто поиграться"
→ **GUI приложение** 🖼️
```powershell
.\run_gui.bat
```

→ **Interactive приложение** 🎮
```powershell
.\build\ascii_rtx_interactive.exe
```

### Случай 3: "Я хочу видеть цифры/статистику"
→ **Batch приложение** 📊
```powershell
.\run.bat
```

### Случай 4: "Я хочу изменять код и быстро тестировать"
→ **Batch приложение** 📊 (редактируйте `src/config/default.cpp`)

### Случай 5: "Я хочу интегрировать в свой код"
→ **Используйте ascii_rtx_engine.lib** 
```cpp
#include "engine/core.h"
#include "config/default.h"
```

## 💾 Сохранение и загрузка конфигурации

### GUI Приложение
- ✅ Конфиг сохраняется **автоматически** при закрытии
- 📄 Файл: `config.ini`
- ⚙️ Загружается при запуске

### Interactive Приложение
- ❌ Конфиг **не сохраняется**
- 📝 Можете редактировать `src/config/default.cpp`

### Batch Приложение
- ❌ Конфиг **не сохраняется**
- 📝 Можете редактировать `src/config/default.cpp`

### Пример config.ini
```ini
[VIDEO]
window_width=1280
window_height=720
fullscreen=false
vsync=true

[ASCII]
char_width=12
char_height=24

[CAMERA]
auto_rotate=true
animation_speed=1.0
```

## 📈 Производительность

### GUI
- FPS: 50-60 (зависит от конфига)
- CPU: 10-20% на современном ПК
- RAM: 50-100 MB

### Interactive
- FPS: 60-70
- CPU: 5-10% на современном ПК
- RAM: 30-50 MB

### Batch
- Кадры/сек: 200-500 (зависит от конфига)
- Время 500 кадров: 5-10 секунд
- RAM: 20-30 MB

## 🔄 Полный цикл работы

```
1. Запустить GUI приложение
   ↓
2. Настроить параметры (разрешение, размер символов)
   ↓
3. Поиграться с WASD управлением
   ↓
4. Сохранить конфиг (автоматически)
   ↓
5. Закрыть приложение
   ↓
6. Запустить еще раз → конфиг загружен!
```


## 🔧 Кастомизация приложений

### GUI
- Отредактируйте `src/apps/gui/imgui_setup.cpp` для добавления новых параметров
- Добавьте параметр в `AppConfig` структуру

### Interactive
- Отредактируйте `src/apps/interactive/input_handler.cpp` для новых горячих клавиш
- Отредактируйте `src/config/default.cpp` для параметров сцены

### Batch
- Отредактируйте `src/apps/viewer/main.cpp` для нового поведения
- Отредактируйте `src/config/default.cpp` для параметров сцены
