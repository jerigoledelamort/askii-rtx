# ASCII RTX v1 - Графический движок для рендеринга в ASCII

## Общее описание

**ASCII RTX** — это экспериментальный графический движок на **C++**, предназначенный для рендеринга 3D сцен в формате ASCII-арта с применением трассировки лучей (raytracing), глобального освещения, временного сглаживания (TAA) и CUDA ускорения.

Проект демонстрирует сложный графический конвейер в условиях экстремальной ограниченности – воспроизведение 3D сцены не пиксельной, а символьной матрицей терминала.

---

## Архитектура проекта

Проект разделён на несколько логических модулей:

### 1. **engine/** – ядро рендеринга

#### `core.cpp/h` – основной класс Engine
- Управляет жизненным циклом сцены и камеры
- Координирует рендеринг кадра через три этапа:
  1. Обновление данных сцены (трансформации объектов)
  2. Вычисление параметров камеры (позиция, направление взгляда, вверх, вправо)
  3. Вызов GPU-рендера для получения буферов люминанса, краёв, RGB и нормалей
- Управляет состоянием рендеринга (аккумулирующие буферы, RNG состояния для выборки)

#### `render.cpp/h` – GPU-рендеринг
- **CUDA ядра** для параллельного трассирования лучей на видеокарте
- Реализует multi-sample raytracing с адаптивной выборкой
- **Основные компоненты:**
  - `trace_scene()` – определение пересечения луча со сценой
  - `compute_shadow()` – вычисление теней с помощью shadow rays
  - `sample_indirect_light()` – непрямое освещение (глобальное освещение через отскоки)
  - `render_frame_buffer()` – главное ядро, вычисляющее RGB для каждого пикселя
  - `saturate01()` – зажимание значения в [0, 1]
  - `luminance_of()` – вычисление яркости по формуле CIE 1931
  - `random_unit_vector()` – генерация случайных единичных векторов (importance sampling)
  - `apply_dither()` – добавление диффузной дизеринговки для уменьшения артефактов
  - `apply_temporal_aa()` – сглаживание по времени между кадрами
- Управление глобальными CUDA буферами и состояниями RNG
- Постобработка: гамма-коррекция, экспозиция, тональное сжатие

#### `camera.cpp/h` – система камеры
- Класс **Camera** – позиция и ориентация в мировом пространстве
- `get_vectors()` – вычисление базисных векторов (forward, right, up)
- `compute_scene_bounds()` – вычисление AABB всех геометрических объектов (оптимизация для автоматической фокусировки)
- `check_collision_camera()` – проверка столкновений камеры со сферами и кубами
- Орбитальная камера с центром в начале координат
- Интегрирована с системой управления (клавиатура, мышь)

#### `scene.cpp/h` – управление сценой
- Хранилище геометрических примитивов: **сферы**, **кубы**, **плоскость**
- `compute_vertex_normals()` – вычисление нормалей в вершинах для сглаживания
- `transform_vertices()` – матричные трансформации (трансляция, поворот по осям XYZ)
- Система анимации: поддержка параметризованного движения объектов во времени
- Данные сцены: массивы параметров (позиция, размер, материал)
- Функция `update()` – обновление трансформаций и анимационных параметров
- Функция `get_data()` – экспорт данных сцены в CUDA-совместимый формат

#### `lighting.cpp/h` – освещение
- `get_light()` – нормализация вектора направления источника света
- `reflect_components()` – вычисление отражённого направления луча относительно нормали (глеевский закон отражения)
- `local_lighting()` – расчёт локального освещения (Phong-модель):
  - **Ambient** – фоновое освещение
  - **Diffuse** – диффузная компонента (Lambert)
  - **Specular** – зеркальная компонента с регулируемым блеском (shininess)
  - Учитывает тени (shadow multiplier)

#### `materials.cpp/h` – система материалов
- Массив предопределённых материалов с параметрами:
  - diffuse, specular, shininess
  - reflectivity, roughness, refractivity
  - IOR (index of refraction), absorption
  - RGB цвет
- **Материалы:**
  - 0 – матовая поверхность (matte)
  - 1 – глянцевый пластик (glossy plastic)
  - 2 – смешанный материал
  - 3 – стекло (glass)
  - 4 – зеркало (mirror)

### 2. **geometry/** – примитивы геометрии

#### `sphere.cpp/h` – сферы
- `hit_sphere()` – пересечение луча со сферой
- Решение квадратного уравнения (дискриминант)
- Возвращает минимальное положительное расстояние t или -1 при отсутствии пересечения

#### `box.cpp/h` – кубы/параллелепипеды
- `hit_box()` – пересечение луча с осью-ориентированным AABB (Axis-Aligned Bounding Box)
- Метод slabs: вычисление пересечения со всеми тремя парами граней
- Возвращает расстояние t или -1 при отсутствии пересечения

#### `plane.cpp/h` – плоскость
- `hit_plane()` – пересечение луча с горизонтальной плоскостью (пол сцены)
- Применяется для создания базовой поверхности

---

### 3. **pipeline/** – постобработка и экспорт

#### `video_ascii.cpp/h` – видеоэкспорт
- Сохранение последовательности рендерированных кадров в видеофайл MP4
- Использует **imageio** для кодирования видео (libx264)
- Преобразование буферов RGB в видеоформат YUV420p
- Параметры: FPS, кодек, разрешение (W×H в пиксельных единицах)

---

### 4. **apps/viewer/** – главное приложение

#### `main.cpp/h` – главный цикл
- **Окружение:** Pygame для SDL2 оконной системы
- **Инициализация:**
  - Выбор разрешения (из предустановок: 640×360 до 2560×1440)
  - Расчёт размера символа для целевого количества символов на экран
  - Загрузка шрифта (Consolas, размер 8–14 пт)
  - Построение rampy характеров (от тёмного к светлому)
  - Инициализация движка рендеринга
- **Главный цикл:**
  - Обработка событий (выход, изменение режима, управление камерой)
  - Рендеринг GPU → RGB буфер
  - Преобразование RGB → ASCII (выборка лучших символов по яркости)
  - Отрисовка ASCII на экран (Pygame surface)
  - Отрисовка UI панели (слайдеры, кнопки, выпадающие меню)
- **Режимы:**
  - **realtime** – живой рендер с ФПС счётчиком
  - **playback** – воспроизведение записанных кадров
- **UI элементы:**
  - Слайдеры для радиуса камеры, высоты, интенсивности света
  - Выпадающее меню для выбора разрешения
  - Кнопки для переключения режимов, сохранения видео
  - Чекбоксы для включения/отключения эффектов (TAA, тени, амбиент)

#### `ui.cpp/h` – система интерфейса
- **Slider** – горизонтальный ползунок с числовым выводом
- **Button** – кнопка с текстовой меткой, обнаружение клика мыши
- **Dropdown** – выпадающее меню для выбора из списка опций
- **Checkbox** – флажок включения/отключения параметров
- Обработка взаимодействия мыши (координаты, клики)
- Рендеринг UI на Pygame surface

---

### 5. **config/** – конфигурация

#### `default.cpp/h` – главный конфиг
- **WINDOW** – разрешение, FPS
- **BAKE** – количество кадров для рендеринга
- **TEMPORAL** – параметры TAA:
  - `base_alpha` – базовое смешивание текущего и предыдущего кадров (0.3 = 30% новый)
  - `motion_scale` – масштабирование влияния движения камеры
  - `base_clamp` – ограничение контрастности между кадрами
- **RENDER** – параметры трассировки:
  - `chars` – набор символов для ASCII арта (от светлого к тёмному: " .,:-~=+*#%@")
  - `samples` – количество лучей на пиксель (2, 4, 8 etc.)
  - `exposure` – экспозиция ( 1.0 = по умолчанию)
  - `gamma` – гамма-коррекция
  - `diffuse_gi_strength` – интенсивность глобального освещения
- **FONT** – параметры шрифта (имя, размер)
- **PERFORMANCE** – целевое количество символов на экран
- **CAMERA** – параметры орбитальной камеры (радиус, высота)
- **LIGHT** – направление источника света
- **LIGHTING** – флаги (амбиент, жёсткие тени)
- **SCENE** – описание сцены (сферы, кубы, анимационные параметры)

---

### 6. **utils/** – вспомогательные функции

#### `char_calibration.cpp/h` – калибровка символов
- `build_char_ramp()` – сортировка набора символов по яркости
- Для каждого символа: рендеринг в буфер, вычисление средней яркости пикселей
- Возвращает строку символов, отсортированную от тёмного к светлому
- Используется для корректного отображения градаций освещённости в ASCII

└─────────────────────────────────────────────────────────────┘
						  ↓
┌─────────────────────────────────────────────────────────────┐
│ 2. ТРАССИРОВКА ЛУЧЕЙ (GPU CUDA)                             │
│    - Генерация первичных лучей из камеры                    │

│    - Локальное освещение (Phong)                            │

│    - Тени (shadow rays)                                      │
└─────────────────────────────────────────────────────────────┘
						  ↓
┌─────────────────────────────────────────────────────────────┐
│ 3. ПОСТОБРАБОТКА (GPU)                                       │
│    - Временное сглаживание (TAA)                            │
│    - Гамма-коррекция                                        │
│    - Экспозиция и тональное сжатие                          │

└─────────────────────────────────────────────────────────────┘
						  ↓
┌─────────────────────────────────────────────────────────────┐
│ 4. ПРЕОБРАЗОВАНИЕ В ASCII (CPU)                             │
│    - Для каждого пикселя: выбор ближайшего символа         │
│    - Сопоставление яркости пикселя с яркостью символа      │
│    - Создание двумерного массива символов                  │
└─────────────────────────────────────────────────────────────┘
						  ↓
┌─────────────────────────────────────────────────────────────┐
│ 5. ОТРИСОВКА НА ЭКРАН (GPU Pygame)                          │
│    - Рендеринг символов на Pygame surface                   │

│    - Отображение на мониторе                                │

## Ключевые особенности

### 1. **Трассировка лучей с CUDA**
- Параллельное вычисление на видеокарте
- Multi-sample anti-aliasing (MSAA)
- Адаптивная выборка пикселей (уменьшение шума)

### 2. **Глобальное освещение (GI)**
- Рекурсивное отслеживание отскочивших лучей
- Diffuse-based indirect lighting
- Контроль интенсивности через `diffuse_gi_strength`

### 3. **Временное сглаживание (TAA)**
- Аккумулирование RGB данных между кадрами
- Динамическое смешивание на основе движения камеры
- Уменьшение мерцания в ASCII режиме

### 4. **Система материалов**
- Поддержка диффузных, зеркальных, отражающих и преломляющих материалов
- Параметризованные свойства для каждого материала

### 5. **Оптимизированная геометрия**
- AABB collision для кубов
- Параметрические примитивы (сферы, кубы, плоскость)
- Аппаратное ускорение на CUDA

### 6. **Интерактивный UI**
- Реал-тайм регулировка параметров
- Орбитальная камера с управлением мышью и клавиатурой
- Выбор разрешения и режимов рендеринга

### 7. **ASCII визуализация**
- Динамическая калибровка символов по яркости
- Адаптивный размер символов для целевого количества на экран
- Дизеринг для улучшения качества

## Параметры конфигурации

| Параметр | Значение | Описание |
|----------|----------|----------|
| `WINDOW.width` | 2560 | Ширина окна в пиксельных единицах |
| `WINDOW.height` | 1440 | Высота окна |
| `WINDOW.fps` | 50 | Целевой FPS |
| `RENDER.samples` | 2 | Сэмплы на пиксель (raytracing) |

| `CAMERA.height` | 1.0 | Высота камеры |

## Применяемые технологии

- **C++** – основной язык разработки
- **CUDA** – GPU-ускорение raytracing
- **Pygame** – оконная система и UI
- **NumPy** – работа с массивами и матрицами
- **imageio & libx264** – экспорт видео
- **Numba** – JIT-компиляция для CPU-функций (опционально)

---

## Заключение

ASCII RTX демонстрирует, как сложные графические техники (raytracing, GI, TAA) могут быть адаптированы к крайне ограниченному формату вывода – текстовой матрице. Проект сочетает низкоуровневую работу с GPU, математическую точность в вычислениях освещения и инженерные решения для оптимизации качества при минимальной информационной плотности символьного буфера.
