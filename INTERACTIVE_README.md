# 🎮 ASCII RTX Interactive - Управление камерой в реальном времени

Интерактивное приложение для визуализации ASCII Ray Tracing с управлением камерой через клавиатуру.

## 🚀 Запуск

### Вариант 1: Через .bat файл (рекомендуется)
```bash
.\run_interactive.bat
```

### Вариант 2: Через PowerShell
```powershell
.\build\ascii_rtx_interactive.exe
```

### Вариант 3: Через CMake
```powershell
cmake --build .\build --config Release
.\build\ascii_rtx_interactive.exe
```

## ⌨️ Управление

### Движение камеры
| Клавиша | Действие |
|---------|----------|
| **W** | Движение вперёд |
| **S** | Движение назад |
| **A** | Движение влево |
| **D** | Движение вправо |
| **Q** | Движение вниз |
| **E** | Движение вверх |

### Вращение камеры
| Клавиша | Действие |
|---------|----------|
| **↑** (Up) | Повернуть вверх (Pitch) |
| **↓** (Down) | Повернуть вниз (Pitch) |
| **←** (Left) | Повернуть влево (Yaw) |
| **→** (Right) | Повернуть вправо (Yaw) |

### Горячие клавиши
| Клавиша | Действие |
|---------|----------|
| **R** | Сбросить позицию камеры |
| **ESC** | Выход из приложения |

## 📊 Интерфейс

Верхняя строка показывает:
- **FPS** - текущая частота кадров
- **Cam** - текущая позиция камеры в 3D пространстве

Основная область - ASCII-визуализация сцены с анимированными объектами.

## ⚙️ Технические характеристики

- **Разрешение ASCII**: 100x28 символов
- **Размер консоли**: 120x40 символов
- **Целевой FPS**: 60 FPS
- **Ввод**: Неблокирующие проверки клавиш через Windows API
- **Рендеринг**: CPU-based (готово к GPU расширению)

## 📝 Архитектура

```
src/apps/interactive/
├── main.cpp            - Главный цикл приложения
├── console_ui.h/cpp    - Управление консолью Windows
└── input_handler.h/cpp - Обработка клавиатурного ввода
```

### Основные компоненты

#### ConsoleUI
Управляет выводом в консоль:
- Очистка экрана
- Позиционирование курсора
- Скрытие/показ курсора
- Рендеринг ASCII кадра

#### InputHandler
Обрабатывает клавиатурный ввод:
- Неблокирующая обработка (GetAsyncKeyState)
- Поддержка одновременного нажатия нескольких клавиш
- Преобразование в команды движения/вращения

#### InteractiveViewer
Главный контроллер приложения:
- Инициализация движка и консоли
- Главный цикл обновления
- Обработка ввода и применение к камере
- RGB → ASCII конвертация
- Расчёт FPS

## 🔧 Расширение функциональности

### Добавление новых клавиш

Отредактируйте `input_handler.cpp`:

```cpp
if (is_key_pressed('X')) {  // Замените 'X' на нужную клавишу
	input.position_delta.z += move_speed;  // Нужное действие
}
```

Доступные коды клавиш:
- `'A'` - `'Z'` - буквы
- `'0'` - `'9'` - цифры
- `VK_UP`, `VK_DOWN`, `VK_LEFT`, `VK_RIGHT` - стрелки
- `VK_SPACE` - пробел
- `VK_ESCAPE` - Escape
- И другие VK_* константы из Windows API

### Изменение скорости движения

В `main.cpp` найдите:

```cpp
const float move_speed = 0.1f;      // Изменить это значение
const float rotate_speed = 0.05f;   // Или это
```

### Изменение ASCII палитры

В `main.cpp`:

```cpp
const char* char_ramp_ = " .:-=+*#%@";  // Замените на собственную
```

Примеры других палиток:
- `" ░▒▓█"` - блочные символы
- `" .:-=+*#%@█"` - расширенная
- `" 123456789"` - числовая

## 🐛 Возможные проблемы и решения

### Консоль закрывается сразу
- Обычно это означает ошибку инициализации
- Проверьте, что исполняемый файл существует: `.\build\ascii_rtx_interactive.exe`
- Пересоберите проект: `cmake --build .\build --config Release`

### Очень низкий FPS
- На старых ПК CPU рендеринг может быть медленным
- Уменьшите размер ASCII (в `main.cpp` измените `ascii_width_` и `ascii_height_`)
- Попробуйте включить видеокарту (редактируйте `render.cpp` для GPU пути)

### Клавиши не работают
- Убедитесь, что консольное окно активно (нажмите на него)
- На некоторых клавиатурах могут быть проблемы с одновременным нажатием - попробуйте по одной

### Картинка мигает
- Это нормально на низких FPS - увеличьте мощность PC или уменьшите разрешение
